# Отчет по выполнению задания «Сетевое взаимодействие программ»

## Описание работы
Целью данной работы было усложнение программного кода шахматного вычислителя угроз с выделением полноценного backend'а (сервера) для решения бизнес-задачи, а также создание сетевого клиента, который отправляет в него данные по сети.
Взаимодействие реализовано на языке **Java** поверх протокола **TCP**.

## Архитектура проекта
Проект построен с использованием системы автоматизации сборки **Gradle** по принципу многомодульного (Multi-project) приложения. 
Он состоит из трех независимых компонентов:
1. `shared` — библиотека бизнес-логики (`ChessMath`), которая не знает про сеть.
2. `server` — приложение-сервер, слушающее TCP порт `9090`. Оно принимает на вход строки с координатами фигур, вызывает `ChessMath` и отправляет текстовый ответ обратно.
3. `client` — консольное приложение, считывающее ввод пользователя и отправляющее его на локальный адрес сервера (`127.0.0.1:9090`).

## Процесс взаимодействия
- Сервер открывает `java.net.ServerSocket(9090)` и находится в режиме бесконечного цикла, принимая входящие подключения (метод `.accept()`).
- Когда клиент вводит координаты в виде строки, разделенной пробелами (например, `4 4 4 8 2 2`), клиент использует `java.net.Socket`, подключается к серверу и передает поток байтов через `PrintWriter`.
- Сервер получает данные, проводит вычисления (какая шахматная фигура угрожает королю) и формирует строку-результат.
- Результат отправляется клиенту обратно по открытому TCP-соединению. Клиент выводит ответ на экран.

## Запуск
Для сборки всех проектов:
```bash
./gradlew build
```
Для старта Backend'а:
```bash
./gradlew :server:run
```
Для старта Клиента:
```bash
./gradlew :client:run -q --console=plain
```
